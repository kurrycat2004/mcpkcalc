Block Layout

/s/BLOCK_LAYOUT/coords/strat

The basic format of the blockLayout is (blockType<:Z|X<-|+>?>?;[x,y,z])

Blocktype

blockType would represent one of the blocktypes listed below.

U can also write the names out

Blocktype blockType
Block b
Cobblewall cw
Fence f
Ladder l
Slab s
Trapdoor td

Orientation

The orientation (X|Z<+|->?) is only important for non-symmetrical blocks like ladder or td.
For blocks like fence and cobblewall it also defines where they are connected to blocks.

Examples:
(cw;[0,0,0]) → A normal cobblewall that is not connected to any block.
(cw:X+;[0,0,0]) → A cobblewall that is connected to a block on the right side (X+).
(f:Z;[0,0,0]) → A fence that is connected to blocks on both Z+ and Z- so it is 0.25b in width and 1b in length.

Coordinates

The coordinates ([x,y,z]) are only whole numbers.

Examples:
[0,0,0], [1,-1,5]

Coords

The basic format of the starting coords is <x>?;z

The coords can be both negative as well as positive decimals.
x being -0.1 would mean that the starting x position is 0.1 from the left edge.
x = 0.1 would mean that the starting x position is 0.1 from the right edge.
The same thing goes for the z coordinate. A negative number means that it's moved by that amount from the back edge and a positive number means that it's moved by that amount from the front edge.

The edges are at -0.3 and 1.3 so -0.1 would mean a z coord of 1.2

Examples:
-0.127 → 0, 1.173
1.2;-0.12 → 0.9, 1.18
